Understand the big picture
Before opening VS Code, understand what you're building

Your app has two separate programs running at the same time, talking to each other. This is called a "full-stack" app — exactly what real companies (Swiggy, Zomato, etc.) build.

DATA FLOW

👤 You
upload CSV
→
⚛ React
port 3000
→
🐍 FastAPI
port 8000
→
🤖 OpenAI
GPT-4o-mini
← Rewritten CSV comes back the same way in reverse

⚛️ Frontend (React) = port 3000

What you see in the browser. The upload button, table preview, download button. It has NO direct access to OpenAI — it only talks to FastAPI.

🐍 Backend (FastAPI) = port 8000

The invisible brain. Receives files, calls OpenAI, sends results back. Your API key lives here — React never touches it directly.

💡
Why two programs? So your secret OpenAI key never goes to the browser (where anyone could steal it). The backend keeps it safe. This is standard architecture — you'll explain it in every interview.

Get OpenAI API key safely
The right way so it never gets exposed again
🔴
Golden Rule: An API key is like a password. It goes ONLY in a .env file. Never in code. Never in chat. Never on GitHub. The .gitignore file already ensures .env is never uploaded.
1
Create the key on OpenAI's website
  • →
    Go to platform.openai.com → sign up/login
  • →
    Click your profile icon (top right) → API Keys
  • →
    Delete the old exposed key (trash icon beside it)
  • →
    Click + Create new secret key → name it rewrite-ai → Create
  • !
    Copy it right now — you can only see it once. It starts with sk-proj-...
  • →
    Click Billing in the left sidebar → add $5 credit. This is enough for hundreds of test rewrites.
2
Save key in the .env file (NOT in the code)

In the backend/ folder there's a file called .env.example. You need to create a copy called .env (no ".example"):

Terminal — inside the backend folder
# Windows: copy .env.example .env # Mac / Linux: cp .env.example .env

Now open backend/.env in VS Code. Replace paste-your-real-key-here with your actual key. Save (Ctrl+S).

backend/.env — what it should look like
OPENAI_API_KEY=sk-proj-your-actual-key-here
✅
.gitignore already has .env listed — so Git will never upload it to GitHub. Your key stays on your computer only.
5
Run the Backend (FastAPI)
Start the Python server — Terminal window #1
💡
What is a "virtual environment" (venv)? A clean isolated Python sandbox just for this project. Without it, packages from different projects collide. Always use one — it's standard practice and interviewers will ask about it.
1
Open VS Code in the project folder
  • →
    Open VS Code → File → Open Folder → select the rewrite-ai folder
  • →
    Open VS Code's built-in terminal: press Ctrl + ` (backtick key — top-left corner, same key as ~). A terminal panel opens at the bottom.
2
Go to the backend folder
VS Code Terminal
# cd = "change directory" — moves you into a folder cd backend # Your prompt should now show something like: # C:\Users\You\rewrite-ai\backend> (Windows) # ~/rewrite-ai/backend $ (Mac)
3
Create and activate the virtual environment
VS Code Terminal
# Create the virtual environment (one-time setup) python -m venv venv # ── Activate it ── # Windows Command Prompt: venv\Scripts\activate # Windows PowerShell (if above fails): venv\Scripts\Activate.ps1 # Mac / Linux: source venv/bin/activate # ✅ Success: your prompt now shows (venv) at the start # Example: (venv) C:\Users\You\rewrite-ai\backend>
⚠️
If PowerShell says "running scripts is disabled", run this first: Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser then try again.
4
Install Python packages
Terminal (must show (venv) in prompt)
# pip reads requirements.txt and installs everything listed # This installs: fastapi, uvicorn, openai, pandas, etc. pip install -r requirements.txt # Wait for it to finish. You'll see lots of text. # Success looks like: "Successfully installed fastapi-0.111.0 ..."
5
Create your .env file with the API key
Terminal
# Windows: copy .env.example .env # Mac / Linux: cp .env.example .env # Open it in VS Code and add your key: code .env

The file opens. Change paste-your-real-key-here to your actual OpenAI key. Save with Ctrl+S.

6
Start the FastAPI server
Terminal
uvicorn main:app --reload # --reload = auto-restarts when you save code changes # # ✅ You should see: # INFO: Uvicorn running on http://127.0.0.1:8000 # INFO: Application startup complete.
✅
Open your browser and go to http://localhost:8000 — you should see {"status":"running"}. Also try http://localhost:8000/docs — this is the auto-generated API documentation.

Leave this terminal window open. The server must keep running while you use the app.

Push your code from the terminal

Open a terminal in the rewrite-ai/ root folder (not backend, not frontend — the main folder). Run these commands one by one:

Terminal — in the rewrite-ai/ root folder
# Initialize Git in this folder git init # Stage all files for commit (.gitignore protects .env automatically) git add . # Save a snapshot with a message git commit -m "Initial commit — ReWrite.AI" # Rename branch to "main" git branch -M main # Connect to GitHub (replace YOUR_USERNAME with your GitHub username) git remote add origin https://github.com/YOUR_USERNAME/rewrite-ai.git # Upload your code git push -u origin main

GitHub may ask for your username and password. For password, use a Personal Access Token (GitHub → Settings → Developer Settings → Personal access tokens → Generate new token).

Deploy live
Frontend → GitHub Pages · Backend → Render.com (both free)
A
Deploy Backend to Render.com
Do this FIRST — you need the URL before deploying frontend
  • 1
    Go to render.com → sign up free (use GitHub login)
  • 2
    Click New + → Web Service
  • 3
    Click Connect a repository → select your rewrite-ai repo
  • 4
    Fill in:
    Name: rewrite-ai-backend Root Directory: backend Runtime: Python 3 Build Command: pip install -r requirements.txt Start Command: uvicorn main:app --host 0.0.0.0 --port $PORT
  • 5
    Scroll to Environment Variables → click Add → Key: OPENAI_API_KEY → Value: your actual key
  • 6
    Click Create Web Service → wait ~3 minutes → you'll get a URL like https://rewrite-ai-backend.onrender.com
  • 7
    Copy that URL — you need it in the next step
B
Deploy Frontend to GitHub Pages
  • 1
    Go to your GitHub repo → click Settings tab
  • 2
    Left sidebar → Pages → Source: select GitHub Actions
  • 3
    Left sidebar → Secrets and variables → Actions → New repository secret:
    Name: VITE_API_URL
    Value: https://rewrite-ai-backend.onrender.com (your Render URL)
  • 4
    Now trigger a deployment: make any tiny change to a file → commit → push to main. The GitHub Actions workflow will automatically build and deploy.
  • 5
    After ~2 minutes, go to repo Settings → Pages → your live URL will be: https://YOUR_USERNAME.github.io/rewrite-ai

Common errors and fixes
Things that trip everyone up
Error You SeeWhat It MeansFix
'python' not found Python not installed or not in PATH Reinstall Python and tick "Add to PATH" checkbox. Try python3 on Mac.
CORS error in browser console React can't reach FastAPI Make sure backend is running (uvicorn). Check the URL in frontend .env is correct.
401 Unauthorized from OpenAI Invalid API key Check your backend/.env file. Make sure you saved it. Re-generate key on OpenAI if needed.
(venv) disappears from prompt Virtual env deactivated Run venv\Scripts\activate again (Windows) or source venv/bin/activate (Mac)
Module not found Package not installed Run pip install -r requirements.txt again with (venv) active
Frontend shows blank page Build or import error Press F12 → Console tab → look for red error. Usually a missing npm install.
Port 8000 already in use Server already running Close any old terminal that was running uvicorn, or use uvicorn main:app --reload --port 8001
💬
Still stuck? Copy the exact error message and ask Claude: "I'm building a React + FastAPI app. I got this error: [paste error]. What does it mean and how do I fix it?" Be specific — include the full error text, not just "it doesn't work."
